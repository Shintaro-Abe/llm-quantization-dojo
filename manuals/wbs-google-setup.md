# WBS 同期のための Google 初期設定ガイド（サービスアカウント方式）

> 対象読者：**初学者のエンジニア**（Google Cloud をはじめて触る人を想定）
> 目的：WBS（進捗表）をリポジトリから Google スプレッドシートへ同期できるように、**自分の手でやる必要がある準備**を済ませる
> 認証方式：**サービスアカウント（SA）方式**（ブラウザログイン不要・現在の推奨）
> 関連：`.steering/20260602-wbs-sync/`（requirements / design / tasklist）

---

## 0. このガイドについて

### なぜ手作業が必要？

WBS 同期の「コードを書く」「設定ファイルを用意する」部分は Claude Code が自動でやります。
でも、**Google Cloud の画面操作**（プロジェクト作成・鍵の発行・スプレッドシートの共有）は、本人がブラウザで行う必要があり、自動化できません。このガイドはその部分だけをまとめたものです。

### このガイドで使う方式（サービスアカウント）

「**サービスアカウント（SA）**」という、人ではなく**プログラム用の Google アカウント**を作ります。SA は `〇〇@〇〇.iam.gserviceaccount.com` というメールアドレスを持ちます。
**そのメールにスプレッドシートを「共有」すると、プログラム（このプロジェクト）からシートを読み書きできる**ようになります。ブラウザでのログイン操作は不要です。

> ℹ️ 以前は「OAuth（ブラウザでログイン）」方式を案内していましたが、現在ツール側の不具合で使えません。詳細は巻末の[付録 A](#付録-aoauth-方式現在使用不可将来用)に残しています。**本文はサービスアカウント方式だけ**で完結します。

### このガイドを終えると、こうなる

- スプレッドシートを読み書きできる **SA の鍵（JSON ファイル）** が手元にある
- WBS 用のスプレッドシートがあり、その **ID（spreadsheetId）** が分かっている
- そのスプレッドシートが **SA に共有**されていて、プログラムから接続できる

### 所要時間とむずかしさ

- だいたい **15〜25 分**。クリック中心で、プログラミング知識はほぼ不要です。

### 用意するもの

- **Google アカウント**（普段使っているもので OK）
- パソコンの**ターミナル**（最後に接続確認のコマンドを 1 つ実行します）

### 大事な約束（セキュリティ）

- このあと発行する **SA の鍵（JSON）は「パスワードと同じ」**です。**人に見せない・リポジトリにコミットしない**。
- 鍵はリポジトリ内の **`secrets/` フォルダ**（Git に含めない設定済み）に置きます。

## ステップ 1：Google Cloud プロジェクトを作る

1. ブラウザで [https://console.cloud.google.com/](https://console.cloud.google.com/) を開き、Google アカウントでログインします。
2. 画面**上部のプロジェクト選択メニュー**（「プロジェクトを選択」と書かれた部分）をクリックします。
3. 右上の **「新しいプロジェクト」** をクリックします。
4. **プロジェクト名**に分かりやすい名前（例：`subbuddy-wbs`）を入れて **「作成」**。
5. 作成後、もう一度上部メニューから**いま作ったプロジェクトを選択**します。

> ✅ 以降の操作は、必ず**このプロジェクトを選んだ状態**で行ってください（画面上部に表示されています）。

---

## ステップ 2：必要な API を有効化する

スプレッドシートを操作するため、2 つの「窓口（API）」をオンにします。

1. 左上の**メニュー（横三本線）** → **「API とサービス」** → **「ライブラリ」** を開きます。
2. 検索ボックスに **`Google Sheets API`** と入力 → 出てきた項目をクリック → **「有効にする」**。
3. 同じ手順で **`Google Drive API`** も検索して **「有効にする」**。

> ✅ これでスプレッドシート（Sheets）とファイル管理（Drive）の窓口が開きました。

---

## ステップ 3：サービスアカウント（SA）を作る

1. 左メニュー → **「IAM と管理」** → **「サービス アカウント」** を開きます。
2. 上部の **「＋ サービス アカウントを作成」** をクリックします。
3. **サービス アカウント名**（例：`wbs-sync`）を入れて **「作成して続行」**。
4. **「ロール（役割）を付与」で、`Service Usage コンシューマー`（`roles/serviceusage.serviceUsageConsumer`）を 1 つだけ追加**します。
   - 役割の検索ボックスに `Service Usage` と入れると候補が出ます。選んで追加し **「続行」**。
   - 理由：SA が（このプロジェクトで）有効化済みの API を呼ぶために必要な**最小権限**です。これが無いと「`Caller does not have required permission to use project`（403）」になります。
   - ※ スプレッドシート自体の読み書き権限は、これとは別に**ステップ 6 の「シート共有」**で与えます。
5. そのまま **「完了」** をクリックします。

> 💡 ロールを付け忘れて作成した場合でも後から追加できます：左メニュー **「IAM と管理」→「IAM」** →（SA を探す or 「アクセスを許可」）→ SA のメールに **「Service Usage コンシューマー」** を付与。

> ✅ 一覧に、作った SA が表示されます。`〇〇@〇〇.iam.gserviceaccount.com` という**メールアドレス**ができていれば成功です（次のステップ以降で使います）。

---

## ステップ 4：SA の鍵（JSON）を作って `secrets/` に置く

1. 一覧から、作った SA の行を**クリック**して開きます。
2. 上部の **「キー」** タブ → **「鍵を追加」** → **「新しい鍵を作成」** をクリックします。
3. 種類は **「JSON」** を選び **「作成」**。→ JSON ファイルが**自動でダウンロード**されます。
4. ダウンロードした JSON ファイルを、このリポジトリの **`secrets/` フォルダ**に移します。
   - ファイル名はそのままで構いません（例：`subbuddy-wbs-1234abcd.json`）。

> ⚠️ この JSON は**パスワードと同じ**です。人に見せない・コミットしない。
> ✅ 確認：鍵が **`git status` に出てこない**こと。`secrets/` 配下は `.gitignore` 済みで自動除外されます（`secrets/.gitkeep` だけが追跡されます）。万一出てくる場合は Claude Code に `.gitignore` 設定を依頼してください。

---

## ステップ 5：WBS 用スプレッドシートを作り、ID を控える

1. [https://sheets.google.com/](https://sheets.google.com/) を開き、**空のスプレッドシートを新規作成**します。
2. 名前を分かりやすく（例：`SubBuddy WBS`）にします。
   - ※ シートの中身（WBS / Archive / Summary タブや見出し・数式）は、このあと Claude Code が自動で用意します。**空のままで大丈夫**です。
3. **URL から `spreadsheetId` を控えます。**`/d/` と `/edit` の間が ID です。
   ```
   https://docs.google.com/spreadsheets/d/【ここが spreadsheetId】/edit
   ```

> ✅ 控えた `spreadsheetId` は、あとで Claude Code に伝えると `wbs/wbs.config.yml`（秘密ではない設定）に記録します。

---

## ステップ 6：スプレッドシートを SA に共有する（最重要）

プログラム（SA）がシートを読み書きできるように、**ステップ 5 のシートを、ステップ 3 の SA のメールに共有**します。

1. スプレッドシート右上の **「共有」** をクリックします。
2. ステップ 3 で作った **SA のメールアドレス（`〇〇@〇〇.iam.gserviceaccount.com`）** を入力して追加します。
3. 権限は **「編集者」** にします。
4. （SA はメール通知を受け取れないので、通知のチェックは外して）**送信／完了** します。

> ⚠️ **この共有を忘れると、プログラムから「権限がない（403）」エラーになります。** 最もよくあるつまずきです。
> 💡 SA のメールが分からなくなったら、「IAM と管理」→「サービス アカウント」の一覧で確認できます。
> 💡 人間にも進捗を見せたい場合は、同じ「共有」で相手のメールを **「閲覧者」** で追加します。
> ⚠️ 共有すると相手に中身が見えます。WBS には**個人情報や秘密情報を載せない**ルールです（開発タスクの予定・進捗だけ）。

---

## ステップ 7：ツール（gws）を入れて接続を確認する

1. **（Claude Code に任せられます）ツール `gws` をインストール**します（Node.js 18 以上が必要）。
   ```
   npm install -g @googleworkspace/cli
   ```
   - コマンド名は `gws`。`gws --version` で確認できます。

2. **鍵の場所を環境変数で指定**します。`.env`（雛形 `wbs/.env.example` は Claude Code が用意）に、ステップ 4 で置いた鍵のパスを書きます。
   ```
   GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE=./secrets/あなたのSA鍵.json
   ```

3. **接続確認**：ステップ 5 の `spreadsheetId` で、シート情報を読めるか試します。Claude Code のセッションでは行頭に `!` を付けて実行できます。
   ```
   ! GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE=./secrets/あなたのSA鍵.json gws sheets spreadsheets get --params '{"spreadsheetId":"あなたのID"}'
   ```
   - スプレッドシートのタイトルなどが JSON で返ってくれば**成功**です。
   - `403`（権限）エラーが出たら、ステップ 6 の**共有**を見直してください。

> 💡 この手順 7 は、鍵と ID を Claude Code に伝えれば**代わりに実行してもらえます**。

## 困ったとき（トラブルシュート）

| 症状 | 対処 |
|---|---|
| メニュー名が手順と違う | Google の画面更新で名称が変わることがあります。**役割が同じもの**を探してください（例：「IAM と管理」内の「サービス アカウント」） |
| **403 "Caller does not have required permission to use project"** | SA に **`Service Usage コンシューマー` ロール**が付いていません（ステップ 3-4）。「IAM と管理」→「IAM」で SA メールに付与してください（反映に数分かかることあり） |
| **権限エラー（403／"The caller does not have permission" / シート関連）** | ① ステップ 6 で**スプレッドシートを SA メールに共有**したか、② ステップ 2 の **API 有効化**が済んでいるか、を確認 |
| **"Requested entity was not found"** | `spreadsheetId` が間違っている、または共有漏れ。ID と共有先メールを再確認 |
| **鍵の作成ができない／ボタンが押せない** | 組織のポリシーで SA 鍵作成が禁止されている場合があります。個人 Google アカウントのプロジェクトでは通常作成できます。会社アカウントの場合は管理者に確認を |
| 鍵が `git status` に出る | `secrets/` に置いたか確認。出る場合は Claude Code に `.gitignore` 設定を依頼 |
| `gws` コマンドが見つからない | `npm install -g @googleworkspace/cli` を再実行。Node.js 18 以上が必要 |

## 付録 A：OAuth 方式（現在使用不可・将来用）

> 🚧 **現在この方式は使えません。** `gws` v0.22.5 の OAuth ブラウザログイン（`gws auth login`）に既知バグがあり、`エラー 400: invalid_request` で失敗します（[Issue #695](https://github.com/googleworkspace/cli/issues/695)。個人 Gmail では [Issue #119](https://github.com/googleworkspace/cli/issues/119) も該当）。**本文のサービスアカウント方式を使ってください。** 以下は将来 `gws` 側が修正されたとき用の参考です。

将来 OAuth を使う場合の流れ（概要）：

1. 「OAuth 同意画面（または Google Auth Platform）」を設定し、ユーザー種類「外部」、自分を**テストユーザー**に追加。
2. 「認証情報」→「OAuth クライアント ID」→ 種類 **「デスクトップ アプリ」**（⚠️「ウェブ アプリケーション」では `invalid_request` になる）で作成し、JSON をダウンロード。
3. その JSON を `~/.config/gws/client_secret.json` に配置。
4. `gws auth login -s sheets,drive`（必要スコープのみ。`recommended` は未検証アプリだと失敗）でブラウザ認証。
5. ログイン情報は `~/.config/gws/` 配下に暗号化（`credentials.enc`）して保存。

> この方式が直ったら、スプレッドシートを SA に共有しなくても自分のアカウント権限で読み書きできるようになります。
