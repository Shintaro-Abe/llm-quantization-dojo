# 進捗管理の始め方（GitHub Issues / Projects）

学習の進捗を **GitHub だけ**で管理します。追加サービスは不要で、学びながら GitHub の実務作法も身につきます。

!!! tip "得たい価値は3つ"
    1. **章の進捗が一覧で見える**（Board ビュー）
    2. **つまずき・所要時間を後から振り返れる**（Issue の記録）
    3. **学習スケジュールをガントチャートで見る**（Roadmap ビュー）

## 全体像

```
[章タスク Issue] ──登録──> [Project]
   学習を記録            ├─ Board   … 進捗の一覧（Backlog/学習中/復習/完了）
   (4点セット/つまずき/時間)  └─ Roadmap … ガントチャート（開始日〜目標日）
```

- **Issue** = 1章ぶんの学習記録（記録の単位）
- **Project** = それを可視化する場所（一覧とガント）

---

## ステップA：章タスク Issue を作る

1. リポジトリの **Issues → New issue** を開く
2. **「章タスク（学習記録）」** テンプレートを選ぶ
3. 章・学習ステップ・つまずき・所要時間を埋めて作成

これで学習の「記録の単位」ができます。章を進めるたびに1つ作るのがおすすめです。

## ステップB：Project を作る

1. リポジトリ上部の **Projects → New project** を開く
2. テンプレートは **「Table」** か **「Board」** から開始（後でビューを足せます）
3. 名前を付けて作成（例：`量子化 学習ロードマップ`）

## ステップC：フィールドを足す（最小構成）

Project の **＋（New field）** から、次の**最小フィールド**を追加します（難易度・定着度などは必要になってから足せばOK）。

| フィールド | 種類 | 用途 |
| :-- | :-- | :-- |
| Status | Single select（既定であり） | Backlog / 学習中 / 復習 / 完了 |
| 章 | Text | どの章か（例「第1章」） |
| 開始日 | Date | ガントの開始点 |
| 目標日 | Date | ガントの終了点 |
| 所要時間(分) | Number | 振り返り用 |

!!! note "章は Text でOK"
    章は **自由記述（Text）** にします。dropdown にすると章が増えるたびに選択肢を直す手間が出るためです。

## ステップD：Board ビュー（進捗の一覧）

1. ビュー追加で **Board** を選ぶ
2. 列（グループ）を **Status** にする
3. カードを Backlog → 学習中 → 復習 → 完了 と動かせば、**今どこまで進んだか**が一目で分かります

## ステップE：Roadmap ビュー（ガントチャート）

1. ビュー追加で **Roadmap** を選ぶ
2. 設定（Date fields）で **開始日＝Start、目標日＝Target** を割り当てる
3. 各 Issue に開始日・目標日を入れると、**章ごとのバーが時間軸に並ぶガントチャート**になります

## ステップF：Issue を Project に追加して紐付ける

1. ステップAで作った Issue を開く
2. 右サイドバーの **Projects** から、作った Project を選ぶ
3. Project 側で Status・章・日付・所要時間を入力

以後は Board で進捗、Roadmap でスケジュールを確認できます。

---

## 内容修正・リンク切れ報告の使い方（鮮度運用）

公式ドキュメントのURLは変わります。学習中に**リンク切れや古い記述**に気づいたら、**「内容修正・リンク切れ報告」** テンプレートで Issue を作ってください。リンク健全性は CI（lychee）でも自動検査しますが、内容の正しさは人の目が要ります。

---

## 付録：GitHub Pages を「GitHub Actions」ソースに切り替える

座学サイトを公開するには、Pages の公開ソースを一度だけ手動で切り替えます（本リポジトリは Actions でビルドして公開する方式です）。

1. リポジトリの **Settings → Pages** を開く
2. **Build and deployment → Source** を **「GitHub Actions」** に変更
3. `main` ブランチに push すると `deploy-docs` ワークフローが走り、完了後に Pages の URL で閲覧できます

!!! warning "公開は Public 前提"
    GitHub Pages での公開は Public リポジトリが前提です。非公開にしたいメモは、公開対象外の `docs/`（ガバナンス文書）側で管理してください。
