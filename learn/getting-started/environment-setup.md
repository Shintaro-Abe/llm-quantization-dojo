# 環境構築（不要です）

このサイトのハンズオンは **Google Colab の無料GPU（T4）** でそのまま動きます。自分のPCにPythonやGPUを用意する必要はありません。まずはここで「開き方」と「GPUの確認方法」だけ押さえましょう。

!!! tip "結論"
    **Googleアカウントでログイン → Colabでノートブックを開く → ランタイムをT4 GPUにする → 上から実行。** これだけです。

## 1. なぜ Colab なのか

- **無料**でGPU（NVIDIA T4・約15GB VRAM）が使える
- **インストール不要**（ブラウザだけ。ライブラリは各ノートブックが自動で入れる）
- 本教材のハンズオンは、この無料T4で**最後まで完走**するよう設計しています

## 2. Google Colab の使い方

### 開く

各章のハンズオンにある **「Open in Colab」バッジ（起動ボタン）** を押すと、その章のノートブックがColabで開きます。

### GPUを有効にする（最初に必ず）

1. メニュー **「ランタイム」→「ランタイムのタイプを変更」** を開く
2. **ハードウェア アクセラレータ** で **「T4 GPU」** を選択して保存

### GPUが効いているか確認する

ノートブックの最初のセルで次を実行し、GPU情報が表示されればOKです。

```python
!nvidia-smi
```

`Tesla T4` と VRAM（約15360MiB）が見えれば成功です。`command not found` や表示なしの場合は、上の手順でT4を選び直してください。

### 実行する

メニュー **「ランタイム」→「すべてのセルを実行」（Run all）** で、上から順に最後まで動きます。

## 3. 無料枠の注意点（挫折しないために）

| 事象 | 対策 |
| :-- | :-- |
| 一定時間でセッションが**切断**される | こまめに実行。切れたら再接続して、セットアップセルから流し直す |
| GPUの**割り当てが得られない**ことがある | 時間を置く／後述のKaggleを使う |
| ランタイムを切ると**ファイルや学習結果は消える** | 必要な成果物はGoogleドライブやローカルに保存する |
| `Out of Memory (OOM)` が出る | より小さいモデルに変える・バッチサイズを下げる（各章のトラブルシュート参照） |

!!! warning "秘密情報を貼らない"
    アクセストークンやAPIキーをノートブックに直書きしないでください。本教材の第1章は**ログイン不要の公開モデル**だけを使うので、トークンは不要です。

## 4. 代替：Kaggle Notebooks（発展・大きめモデル向け）

より大きいモデル（7B級など）を試したいときは **Kaggle Notebooks** が有力です。

- 無料で **T4 GPU ×2（合計約32GB VRAM）**、週あたりの利用時間に上限あり
- 利用には Kaggle アカウントと、電話番号認証（GPU有効化）が必要

第1章は Colab だけで完走できます。Kaggle は7B級に挑む**発展課題**で案内します。
