# プロダクト要求定義書（product-requirements.md）

- 最終更新日: 2026-06-13
- ステータス: ドラフト（承認待ち）
- 関連: [事前調査 research/](../research/README.md) ／ [採用構成A案](../research/06-options-and-recommendation.md)

## 1. プロダクトビジョンと目的

**ビジョン**: 初学者のAIエンジニアが、LLMの量子化手法を「迷わず」学び、学習後に**現場の即戦力**になれる個人学習ツールを提供する。

**目的**:
- 量子化の主要手法（NF4/QLoRA・GGUF・GPTQ・AWQ ほか）を、概念理解→公式情報→手を動かすハンズオン→定着まで一気通貫で学べるようにする。
- 初学者が最もつまずく「環境構築」を排除し、ブラウザだけで学習を始められるようにする。
- できる限り**無料**のサービスのみで運用し、**公式かつ最新**の情報に追従しやすい構造にする。

本ツールは**個人学習用**であり、商用配布やチーム運用は対象外とする。

## 2. ターゲットユーザーと課題・ニーズ

**主対象**: LLMの量子化を初めて学ぶAIエンジニア（Pythonは書けるが量子化は未経験）。

| 課題 | 具体例 | 本ツールでの解消 |
| :-- | :-- | :-- |
| 何から学べばよいか分からない | 手法が多く全体像が掴めない | 学習順序を固定したカリキュラム（章立て） |
| 環境構築で挫折する | CUDA/依存関係/GPU確保で前に進めない | Colabワンクリック導線（環境構築不要） |
| 情報が古い/断片的 | 記事のバージョン差・リンク切れ | 薄い自作解説＋公式ドキュメントへのリンク集＋鮮度管理 |
| 学んだ気で終わる | 手を動かさず定着しない | ハンズオンNotebook＋理解度確認＋進捗管理 |
| 実務に繋がらない | 学習が現場作業と乖離 | 実務で使うライブラリ・標準手法を題材にする |

## 3. 主要な機能一覧（4本柱）

1. **座学ドキュメント**: 各手法の概念を平易に解説するWebサイト（公式ドキュメントへの導線を含む）。
2. **ハンズオン教材**: 手を動かして学ぶJupyter Notebook（Python）。
3. **ハンズオン実行環境**: Google Colab（無料GPU）をワンクリックで起動。ローカル実行（llama.cpp）は発展で扱う。
4. **学習内容/進捗管理の外部連携**: GitHub Issues（章タスク）／Projects（ロードマップ・定着度）で進捗を管理。

## 4. 成功の定義（ゴール指標）

- 学習者が**ローカル環境構築なし**で第1章のハンズオンを最後まで完走できる。
- 「概念→公式ドキュメント→ハンズオン→理解度確認」を迷わず一直線にたどれる。
- カリキュラムを修了した学習者が、実務で量子化済みモデルの取り扱い・基本的な量子化作業に着手できる状態になる。
- 運用コストが原則0円（無料枠の範囲）で維持できる。

## 5. ビジネス要件（運用方針）

- **コスト**: 原則無料。MkDocs＋GitHub Pages＋GitHub Actions＋Colab/Kaggleの無料枠で完結。
- **公開**: GitHubリポジトリはPublic（GitHub Pages無料公開・ポートフォリオ化）。
- **保守性**: 公式情報の変化に追従しやすいよう、本文は薄く・リンク集と鮮度管理で陳腐化を抑える。
- **拡張性**: 章を増やすだけでカリキュラムを拡張できる構造。Anki(SRS)やNotion連携は後付け可能にする。

## 6. ユーザーストーリー

- US-1: 初学者として、環境構築をせずにブラウザだけでハンズオンを始めたい。挫折せず最初の一歩を踏み出すため。
- US-2: 初学者として、各手法の「何のため・いつ使う」を短時間で理解したい。全体像を掴むため。
- US-3: 初学者として、公式の最新情報に正しくたどり着きたい。古い情報で誤解しないため。
- US-4: 学習者として、自分の進捗と理解度を記録したい。継続と振り返りのため。
- US-5: 学習者として、学んだ用語を定着させたい。実務で迷わず使うため。

## 7. 受け入れ条件（MVP=第1章 bitsandbytes NF4/QLoRA）

- AC-1: 第1章ページに概念解説と公式ドキュメントへのリンク集（最終確認日付き）がある。
- AC-2: 第1章Notebookに「Open in Colab」バッジがあり、無料GPU（T4）で**Run allが完走**する（NF4で4bitロード→QLoRA微調整→推論の前後比較まで）。
- AC-3: 座学サイトがGitHub Pagesで公開され、第1章へ到達できる。
- AC-4: 章タスク用Issueテンプレートと、Projectで進捗管理を始める手順書がある。
- AC-5: 既定モデルは無料T4で確実に完走するサイズ（1〜3B級・ログイン不要な非ゲート/Apache・MIT推奨）。7Bは発展課題として案内する。

## 8. 機能要件

| ID | 要件 |
| :-- | :-- |
| FR-1 | 学習順序を定義したカリキュラム（章立て）を提供する：NF4/QLoRA → GGUF → GPTQ → AWQ →（発展）HQQ/AutoRound/SmoothQuant/FP8 |
| FR-2 | 各章は「概念 → 公式ドキュメント → ハンズオンNotebook → 理解度確認/タスク(Issue)」の4点セットで構成する |
| FR-3 | 各Notebookに Colab 起動バッジを付与し、依存はバージョン固定（pin）する |
| FR-4 | 公式ドキュメント・論文へのリンク集を章ごとに持ち、最終確認日を記録する |
| FR-5 | 章タスク・内容修正用のIssueテンプレート、PRテンプレートを提供する |
| FR-6 | GitHub Projects（Board/Roadmap、章・定着度等のフィールド）で進捗管理を行う手順を提供する |

## 9. 非機能要件

| ID | 区分 | 要件 |
| :-- | :-- | :-- |
| NFR-1 | コスト | 運用は無料枠で完結する |
| NFR-2 | 環境 | 学習者はローカル環境構築なしで開始できる（Colab前提） |
| NFR-3 | 鮮度 | 公式情報に追従しやすく、リンク切れを検出できる（CI＋最終確認日） |
| NFR-4 | 互換 | 教材は無料GPU（Colab T4 15GB / Kaggle T4×2）で動作する範囲で設計する |
| NFR-5 | 言語 | 本文は日本語。コード・専門用語は英語表記 |
| NFR-6 | 保守 | 1リポジトリでGit一元管理し、変更履歴を追跡できる |
| NFR-7 | セキュリティ | 認証情報・PIIをリポジトリに含めない（gitleaks運用） |

## 10. スコープ外（本フェーズ）

- 大規模モデル（30B/70B）の学習・量子化（無料枠外）。
- 有料コース連携、チーム/組織運用、認定証発行。
- Anki/Notion/Sheets への自動連携の実装（設計上の後付け余地のみ確保）。
