---
description: WBS spec(wbs.yml) を Google スプレッドシートへ同期（差分提示→確認→反映）
---

WBS 同期を実行する。**スプレッドシートへ書き込む前に必ずユーザーへ確認する（確認ゲート）。承認なしに反映してはならない。**

手順：

1. **ドライランで差分を取得**する（副作用なし）：
   ```
   npm --prefix wbs run sync
   ```
   出力の「追加 / 更新 / 変更なし / 退避」の件数と内容を読む。

2. **差分がない場合**（追加 0・更新 0・退避 0）：
   「差分はありません。同期は不要です」と伝えて終了する（apply しない）。

3. **差分がある場合**：`AskUserQuestion` でユーザーに同期の可否を確認する。
   - 質問文に **追加 / 更新 / 退避の件数**と、主な対象（最大 10 件程度の WBS ID・タスク名・変更内容）を含める。
   - 選択肢は「反映する」「やめる」の 2 つ。

4. **「反映する」が選ばれたときのみ** apply する：
   ```
   npm --prefix wbs run sync:apply
   ```
   実行後、反映行数・退避件数を報告する。

5. **「やめる」なら**何も書き込まずに終了する。

注意：
- ユーザーの承認なしに `sync:apply` を実行しない（要件 AC-2b）。
- 認証は `wbs/.env` のサービスアカウント方式（`GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE`）。
- 失敗時はエラー出力をそのまま提示し、握りつぶさない（例：シート未共有なら 403）。
- このコマンドは手動（`/wbs-sync`）でも、tasklist 完了フックからの提案経由でも同じ流れで使う。
